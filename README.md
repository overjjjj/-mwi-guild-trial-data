# Milky Way Idle 公会试炼分配插件使用说明

适用版本：

- 会长端 `milkyway-guild-trial-allocator.user.js` v0.13.0
- 会员端 `milkyway-guild-trial-member.user.js` v0.3.0
- 远程服务 `https://mwi-guild-trial-data.vercel.app`

## 1. 功能范围

本插件用于辅助公会试炼的信息采集、成员能力汇总和报名方向分配。

- 每周读取 4 场生活试炼和 2 场战斗试炼。
- 读取公会成员、公开等级和当前报名方向。
- 会员主动上传自己的生活等级、战斗等级、装备、能力和房屋摘要。
- 根据当周 Boss 类型和属性权重生成战斗分配。
- 支持将早期随意报名视为“乱报名”，重新规划全部成员。
- 会长发布方案后，会员只能读取自己的正式分配。

插件不会直接替成员报名，不会绕过游戏权限，也不会使用食物和饮料数据。

## 2. 系统组成

```text
会员端油猴脚本
    | 主动上传角色摘要
    v
Vercel 服务
    | 使用服务端 GitHub PAT
    v
GitHub 私有仓库中的明文 JSON
    ^
    | 拉取资料、发布方案
会长端油猴脚本
```

正式服务地址：

```text
https://mwi-guild-trial-data.vercel.app
```

健康检查：

```text
https://mwi-guild-trial-data.vercel.app/v1/health
```

正常响应示例：

```json
{"ok":true,"weekId":"2026-07-17"}
```

## 3. 安装脚本

### 3.1 前置条件

1. 浏览器安装 Tampermonkey 或兼容的油猴扩展。
2. 会长安装会长端脚本。
3. 普通成员安装会员端脚本。
4. 使用脚本支持的 Milky Way Idle 正式或测试域名。

支持域名：

- `https://www.milkywayidle.com`
- `https://test.milkywayidle.com`
- `https://www.milkywayidlecn.com`
- `https://test.milkywayidlecn.com`

### 3.2 会长端

安装文件：`milkyway-guild-trial-allocator.user.js`

安装后，游戏页面右下角会出现“公会试炼分配助手”按钮。

### 3.3 会员端

安装文件：`milkyway-guild-trial-member.user.js`

安装后，游戏页面右下角会出现“公会试炼会员端”按钮。

## 4. 会长端首次设置

打开会长端面板，进入“设置”页，填写：

| 设置项 | 填写内容 |
| --- | --- |
| 远程服务地址 | `https://mwi-guild-trial-data.vercel.app` |
| 公会 ID | 自定义稳定 ID，例如 `guild-cn-1` |
| 会长令牌 | Vercel 中配置的 `LEADER_TOKEN` |

公会 ID 只能包含字母、数字、下划线和连字符。设置后不要每周更换，否则会写入不同的数据目录。

填写完成后点击“保存设置”。

### 4.1 名额设置

“生活试炼默认名额”和“战斗试炼默认名额”只在页面未读取到实际名额时使用。正常情况下应优先读取游戏试炼页面显示的实际名额。

### 4.2 报名处理模式

根据公会当前情况二选一：

**重新规划全部报名**

勾选“重新规划全部报名”。适用于尚未确定方向、成员已经随意报名的情况。

- 当前报名不占最终名额。
- 所有成员重新进入全局优化。
- 输出会提示需要改报的成员。

**只分配未报名成员**

取消“重新规划全部报名”，并勾选“只给未报名成员分配”。适用于已经确认的报名不应被改动的情况。

- 已报名成员保留当前方向。
- 只给游戏页面识别到的“未参加/未报名”成员分配。

## 5. 同步会员名单

1. 打开游戏公会“成员”页面。
2. 点击会长端顶部“读取成员/等级”。
3. 检查设置页“成员 CSV”中是否包含成员姓名。
4. 确认远程服务地址、公会 ID 和会长令牌已保存。
5. 打开公会“试炼”页面并读取完整的 4 场生活和 2 场战斗试炼。
6. 点击“同步试炼和会员名单”。

同步后，成员 CSV 中去重后的姓名会写入本周配置。会员只有在角色名与名单匹配时才能上传和读取分配。

## 6. 会员端使用

会员首次使用时：

1. 刷新游戏页面，等待角色数据加载完成。
2. 打开“公会试炼会员端”。
3. 服务地址填写 `https://mwi-guild-trial-data.vercel.app`。
4. 公会 ID 填写会长公布的公会 ID，例如 `guild-cn-1`。
5. 点击“保存设置”。
6. 点击“读取本角色”。
7. 检查页面显示的战斗等级、装备数量和生活专业等级。
8. 点击“上传并刷新”。

上传成功后会显示服务器收到资料的时间。

### 6.1 每周更新

角色等级、装备、能力或配装变化后，再次执行：

1. “读取本角色”
2. “上传并刷新”

只想查看会长是否发布新方案时，点击“仅刷新分配”，不需要重复上传。

### 6.2 会员可看到的内容

- 本周生活试炼个人适配排名。
- 本周战斗试炼个人适配排名。
- 会长发布给自己的生活和战斗正式分配。
- 正式方案发布时间。

会员端不能读取其他成员的正式分配。

## 7. 会长每周操作流程

公会周在每周五 `00:00 UTC` 重置。建议按以下顺序操作。

### 第一步：读取本周试炼

1. 打开公会“试炼”页面。
2. 确保页面完整显示 4 场生活试炼和 2 场战斗试炼。
3. 点击“读取页面”。
4. 检查面板显示的试炼名称、名额和已报名人数。

如果提示“生活 x/4、战斗 x/2”，说明读取不完整。留在试炼页面刷新后重新读取，不要直接生成方案。

### 第二步：读取成员

1. 打开公会“成员”页面。
2. 点击“读取成员/等级”。
3. 检查成员 CSV 中的姓名、等级和当前报名方向。

游戏没有推送的公开数据不会被脚本猜测。会员主动上传或手工导入的数据优先用于补全。

### 第三步：同步试炼配置

点击“同步试炼和会员名单”。

同步内容包括：

- 当周 4 场生活试炼。
- 当周 2 场战斗遭遇。
- 实际名额和当前报名人数。
- Boss 属性权重。
- 成员 CSV 中的本周会员姓名名单。

同步后，会员点击“仅刷新分配”即可看到个人适配排名；此时还不代表正式分配。

### 第四步：拉取会员上传

点击“拉取会员上传”。

远程资料会合并进成员 CSV，包括：

- 战斗和专业等级。
- 武器和伤害类型摘要。
- 非生产装备与强化等级。
- 战斗能力和触发器摘要。
- 战斗房屋等级摘要。

手工填写的评分不会被自动上传覆盖。

### 第五步：补充缺失资料

可选择以下方式：

- 打开成员公开资料，在“概览”或“专业”页点击“读取当前资料”。
- 粘贴角色战斗导出 JSON，点击“导入上述 JSON”。
- 使用兼容脚本的公开导出接口，点击“一键读取并导入本角色”。
- 直接编辑成员 CSV 中的评分和偏好。

角色导出没有名字时，必须填写“角色导出绑定成员名”。

### 第六步：生成分配

1. 确认报名处理模式。
2. 确认 Boss 属性权重 JSON 没有解析错误。
3. 点击“生成分配”。
4. 检查每场试炼人数、未分配成员和需要改报的成员。
5. 点击“复制结果”可发送到公会群。

需要固定减益角色时，在成员 CSV 中把 `role` 填为 `debuff` 或“减益”，并把 `fixedCombat` 填为本周 Boss 的英文 key 或中文名。固定成员会先占位，剩余名额再分配；目标未出现或名额不足时，该成员会显示为未分配，不会改派另一场。

生活自动读取的数值是专业等级，只能作为有效产出的代理。也可以把对应专业列改填每小时有效工作量。生活和战斗的剩余成员都使用贪心近似 Leximin，优先提高当前最弱组的 `总评分 / (1 + 0.01 × 人数)`；这是快速均衡算法，不是枚举全部组合的精确全局解。

战斗分配会根据当周遭遇类型使用不同权重：

| 遭遇 | 默认侧重 |
| --- | --- |
| 獾 | 魔法、单体、低魔法闪避目标 |
| 变色龙 | 物理、单体 |
| 水母 | 魔法、续航、治疗和辅助 |
| 刺猬 | 魔法、单体、续航 |
| 虫群 | AOE、混合伤害、坦克、治疗和辅助 |

算法还会检查新增参与者的边际贡献是否足以抵消每人带来的 `1%` 怪物 HP 增长。

### 第七步：发布正式方案

确认方案后点击“发布当前方案”。

只有已经上传过资料、且能与远程成员 ID 匹配的会员会收到个人正式分配。发布后通知会员点击“仅刷新分配”。

## 8. 成员 CSV

CSV 第一行必须是表头，`name` 必填。常用字段：

| 类别 | 字段 |
| --- | --- |
| 基础 | `name`, `level`, `lifeLevel`, `combatLevel` |
| 当前报名 | `currentLife`, `currentCombat` |
| 生活评分 | `milking`, `foraging`, `woodcutting`, `cheesesmithing`, `crafting`, `tailoring`, `cooking`, `brewing`, `alchemy`, `enhancing` |
| 战斗评分 | `combat`, `aoe`, `single`, `physical`, `magic`, `stab`, `slash`, `blunt`, `ranged`, `tank`, `healer`, `sustain`, `support`, `power` |
| 分类 | `role`, `damageType`, `fixedCombat` |
| 偏好 | `preferLife`, `preferCombat`, `avoid` |

评分数值越高，分配优先级越高。

`preferLife` 和 `preferCombat` 可以填写英文 key 或中文试炼名。`avoid` 可使用英文逗号或竖线分隔多个不想参加的试炼。

`role` 支持 `debuff` 或“减益”。`fixedCombat` 是硬约束，可填 `badger`、`chameleon`、`jellyfish`、`hedgehog`、`swarm` 或对应中文名；固定位置优先于 `avoid`。

## 9. 数据与安全

### 9.1 上传内容

会员端上传经过裁剪的角色摘要，不上传：

- 游戏登录凭据。
- 食物和饮料。
- 背包全部物品。
- 成就和无关角色数据。

### 9.2 存储方式

会员资料以明文 JSON 保存到 GitHub 私有仓库：

```text
guilds/<公会ID>/weeks/<周ID>/members/<成员ID>.json
```

拥有该私有仓库读取权限的人可以看到这些资料。

### 9.3 令牌规则

- `GITHUB_TOKEN` 只保存在 Vercel 环境变量中，不进入油猴脚本。
- `LEADER_TOKEN` 只交给会长和授权管理人员。
- 会员端不使用令牌，只按公会 ID 和当前角色名匹配本周名单。
- PAT 或会长令牌泄露后应立即更换。

姓名匹配不是强身份认证。懂接口的人可以伪造名单中的角色名并覆盖该角色资料，也可以读取该角色的正式分配。需要防止冒用时，应恢复签名会员令牌或接入游戏官方可验证身份。

更换 Vercel 环境变量后必须重新部署，旧部署不会自动获得新值。

## 10. 常见问题

### 10.1 “尚未捕获登录角色数据”

刷新游戏页面，等待角色完全加载后再点击“读取本角色”。如果仍失败，可使用兼容脚本导出或粘贴角色 JSON。

### 10.2 读取到的试炼与实际不一致

必须停留在公会“试炼”页面，并确保 4 场生活和 2 场战斗都已加载。刷新页面后重新点击“读取页面”。

### 10.3 成员等级读取不到

- 先打开公会“成员”页面再点击“读取成员/等级”。
- 公开资料需要实际打开成员资料页，再点击“读取当前资料”。
- 隐藏资料或游戏未推送的字段需要会员端主动上传。

### 10.4 会员提示“不在同步的公会名单中”

检查会员端公会 ID 是否与会长端一致，并确认会长已在本周读取成员 CSV 后点击“同步试炼和会员名单”。姓名匹配忽略大小写和空格，但不会猜测不同拼写。

### 10.5 会长操作返回 401

检查会长端设置中的令牌是否与 Vercel `LEADER_TOKEN` 一致。不要填写 GitHub PAT。

### 10.6 GitHub 写入返回 403

检查 fine-grained PAT：

- Resource owner 正确。
- 只选择仓库 `-mwi-guild-trial-data`。
- `Contents` 权限为 `Read and write`。
- PAT 未过期、未撤销。

### 10.7 健康检查正常，但修改变量后功能仍旧异常

Vercel 环境变量只对新部署生效。修改变量后执行 Redeploy，再测试正式域名。

### 10.8 请求跳转到 Vercel 登录页

项目启用了 Vercel SSO Deployment Protection。油猴脚本无法完成交互式 Vercel 登录，需要关闭该项目的 SSO 部署保护。应用自身的会员和会长 Bearer Token 鉴权仍会保留。

### 10.9 生成方案后大量成员未分配

依次检查：

1. 成员 CSV 是否有成员姓名。
2. 页面是否读取到实际试炼名额。
3. 是否错误勾选“只给未报名成员分配”。
4. 会员上传是否已拉取。
5. `avoid` 是否排除了过多试炼。
6. Boss 属性 JSON 是否能正常解析。

## 11. 推荐的公会管理流程

```text
周五重置后读取试炼
  -> 读取成员和当前报名
  -> 同步试炼和会员名单
  -> 会员上传本周角色快照
  -> 会长拉取会员上传
  -> 补充缺失资料
  -> 生成并审核分配
  -> 发布正式方案
  -> 会员刷新查看个人分配
  -> 按推荐方向在游戏中手工报名
```

插件只提供辅助推荐和信息汇总，最终报名与调整仍由公会管理人员和成员在游戏内完成。
