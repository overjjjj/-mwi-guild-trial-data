# Milky Way Idle 公会试炼分配插件使用说明

适用版本：

- 会长端 `milkyway-guild-trial-allocator.user.js` v0.17.0
- 会员端 `milkyway-guild-trial-member.user.js` v0.6.0
- 远程服务 `https://mwi-guild-trial-data.vercel.app`

## 1. 功能范围

本插件用于辅助公会试炼的信息采集、成员能力汇总和报名方向分配。

- 每周读取 4 场生活试炼和 2 场战斗试炼。
- 读取公会成员、公开等级和当前报名方向。
- 会员主动上传自己的生活等级、战斗等级、装备、能力和房屋摘要。
- 根据当周首领类型和属性权重生成战斗分配。
- 结合当前武器职业和实际技能推荐战斗技能；治疗只计入自然法师。
- 支持将早期随意报名视为“乱报名”，重新规划全部成员。
- 会长发布方案后，会员只能读取自己的正式分配。

插件不会直接替成员报名，不会绕过游戏权限，也不会使用食物和饮料数据。

## 最简使用

会长每周只需跟随面板顶部的蓝色主按钮：

1. 按提示依次完成“读取试炼”“读取成员”“同步本周名单”和“拉取会员资料”。
2. 在“成员”页只处理带橙色提示的成员，给减益、坦克等特殊成员选择职责或固定战斗位置。
3. 继续点击主按钮生成分配并发布方案。

“分配”页顶部会显示试炼、成员、职业、技能、上传和需处理人数。点击有问题的项目会直接进入对应成员筛选。成员页默认勾选“只看需处理”，取消勾选即可查看完整名单。

会员首次展开“连接设置”并保存服务地址和公会编号，以后每周只需点击“一键上传并查看分配”。只查看会长的新方案时点击“只查看最新分配”。

成员原始表格、角色原始数据和首领权重均放在“高级维护”中，正常使用无需展开。

## 2. 系统组成

```text
会员端油猴脚本
    | 主动上传角色摘要
    v
Vercel 服务
    | 使用服务端 GitHub PAT
    v
GitHub 私有仓库中的明文 JSON
    ^
    | 拉取资料、发布方案
会长端油猴脚本
```

正式服务地址：

```text
https://mwi-guild-trial-data.vercel.app
```

健康检查：

```text
https://mwi-guild-trial-data.vercel.app/v1/health
```

正常响应示例：

```json
{"ok":true,"weekId":"2026-07-17"}
```

## 3. 安装脚本

### 3.1 前置条件

1. 浏览器安装 Tampermonkey 或兼容的油猴扩展。
2. 会长安装会长端脚本。
3. 普通成员安装会员端脚本。
4. 使用脚本支持的 Milky Way Idle 正式或测试域名。

支持域名：

- `https://www.milkywayidle.com`
- `https://test.milkywayidle.com`
- `https://www.milkywayidlecn.com`
- `https://test.milkywayidlecn.com`

### 3.2 会长端

安装文件：`milkyway-guild-trial-allocator.user.js`

安装后，游戏页面右下角会出现“公会试炼分配助手”按钮。
脚本包含 GitHub Raw 自动更新地址，Tampermonkey 可按扩展自身的检查周期获取新版本。

### 3.3 会员端

安装文件：`milkyway-guild-trial-member.user.js`

安装后，游戏页面右下角会出现“公会试炼会员端”按钮。
脚本包含 GitHub Raw 自动更新地址。

## 4. 会长端首次设置

打开会长端面板，进入“连接”页，填写：

| 设置项 | 填写内容 |
| --- | --- |
| 服务地址 | `https://mwi-guild-trial-data.vercel.app` |
| 公会编号 | 自定义稳定编号，例如 `guild-cn-1` |
| 会长令牌 | Vercel 中配置的 `LEADER_TOKEN` |

公会编号只能包含字母、数字、下划线和连字符。设置后不要每周更换，否则会写入不同的数据目录。

填写完成后点击“保存连接”。

### 4.1 名额设置

“生活试炼默认名额”和“战斗试炼默认名额”只在页面未读取到实际名额时使用。正常情况下应优先读取游戏试炼页面显示的实际名额。

### 4.2 报名处理模式

根据公会当前情况二选一：

**重新规划全部报名**

勾选“重新规划全部报名”。适用于尚未确定方向、成员已经随意报名的情况。

- 当前报名不占最终名额。
- 所有成员重新进入全局优化。
- 输出会提示需要改报的成员。

**只分配未报名成员**

取消“重新规划全部报名”，并勾选“只给未报名成员分配”。适用于已经确认的报名不应被改动的情况。

- 已报名成员保留当前方向。
- 只给游戏页面识别到的“未参加/未报名”成员分配。

## 5. 同步会员名单

1. 打开游戏公会“成员”页面。
2. 点击会长端顶部主按钮显示的“读取成员”。
3. 在“成员”页确认成员姓名已经出现。
4. 确认服务地址、公会编号和会长令牌已保存。
5. 打开公会“试炼”页面并读取完整的 4 场生活和 2 场战斗试炼。
6. 点击“同步本周名单”。

同步后，成员资料中去重后的姓名会写入本周配置。会员只有在角色名与名单匹配时才能上传和读取分配。

## 6. 会员端使用

会员首次使用时：

1. 刷新游戏页面，等待角色数据加载完成。
2. 打开“公会试炼会员端”。
3. 展开“连接设置”。
4. 服务地址填写 `https://mwi-guild-trial-data.vercel.app`，公会编号填写会长公布的编号。
5. 点击“保存连接”。
6. 点击“一键上传并查看分配”。
7. 检查页面显示的角色资料、个人适配和正式分配。

上传成功后会显示服务器收到资料的时间。

### 6.1 每周更新

角色等级、装备、技能或配装变化后，再次点击“一键上传并查看分配”。只想查看会长是否发布新方案时，点击“只查看最新分配”，不需要重复上传。

### 6.2 会员可看到的内容

- 本周生活试炼个人适配排名。
- 本周战斗试炼个人适配排名。
- 会长发布给自己的生活和战斗正式分配。
- 正式方案发布时间。

会员端不能读取其他成员的正式分配。

## 7. 会长每周操作流程

公会周在每周五 `00:00 UTC` 重置。建议按以下顺序操作。

### 第一步：读取本周试炼

1. 打开公会“试炼”页面。
2. 确保页面完整显示 4 场生活试炼和 2 场战斗试炼。
3. 点击顶部主按钮显示的“读取试炼”。
4. 检查面板显示的试炼名称、名额和已报名人数。

如果提示“生活 x/4、战斗 x/2”，说明读取不完整。留在试炼页面刷新后重新读取，不要直接生成方案。

### 第二步：读取成员

1. 打开公会“成员”页面。
2. 点击顶部主按钮显示的“读取成员”。
3. 在“成员”页处理默认筛出的异常成员；需要检查所有人时取消“只看需处理”。

游戏没有推送的公开数据不会被脚本猜测。会员主动上传或手工导入的数据优先用于补全。

### 第三步：同步试炼配置

在“连接”页点击“同步本周名单”。

同步内容包括：

- 当周 4 场生活试炼。
- 当周 2 场战斗遭遇。
- 实际名额和当前报名人数。
- 首领属性权重。
- 成员资料中的本周会员姓名名单。

同步后，会员点击“只查看最新分配”即可看到个人适配排名；此时还不代表正式分配。

### 第四步：拉取会员资料

在“成员”页点击“拉取会员资料”。

远程资料会合并进成员资料，包括：

- 战斗和专业等级。
- 武器和伤害类型摘要。
- 非生产装备与强化等级。
- 战斗能力和触发器摘要。
- 战斗房屋等级摘要。

手工填写的评分不会被自动上传覆盖。

### 第五步：补充缺失资料

可选择以下方式：

- 在“成员”页打开公开资料后点击“读取已打开的资料”。
- 在“高级维护”中粘贴角色导出内容，点击“导入粘贴内容”。
- 在“高级维护”中直接编辑成员原始表格。

角色导出没有名字时，必须在高级维护中填写“绑定成员名”。

### 第六步：生成分配

1. 在“连接”页确认报名处理模式。
2. 在“成员”页为特殊成员选择职责、固定战斗或偏好。
3. 点击“生成分配”。
4. 检查每场试炼人数、技能建议、未分配成员和需要改报的成员。
5. 点击“复制结果”可发送到公会群。

需要固定减益角色时，在“成员”页直接选择“职责：减益”和“固定：对应试炼”。固定成员会先占位，剩余名额再分配；目标未出现或名额不足时，该成员会显示为未分配，不会改派另一场。

生活自动读取的数值是专业等级，只能作为有效产出的代理。也可以把对应专业列改填每小时有效工作量。生活和战斗的剩余成员都使用贪心近似 Leximin，优先提高当前最弱组的 `总评分 / (1 + 0.01 × 人数)`；这是快速均衡算法，不是枚举全部组合的精确全局解。

战斗分配会根据当周遭遇类型使用不同权重：

| 遭遇 | 默认侧重 |
| --- | --- |
| 獾 | 魔法、单体、低魔法闪避目标 |
| 变色龙 | 物理、单体 |
| 水母 | 魔法、续航、治疗和辅助 |
| 刺猬 | 魔法、单体、续航 |
| 虫群 | 群体攻击、混合伤害、坦克、治疗和辅助 |

算法还会检查新增参与者的边际贡献是否足以抵消每人带来的 `1%` 怪物 HP 增长。

战斗结果会显示最多 3 个技能建议。建议优先使用会员上传的实际技能、触发目标和当前武器职业：虫群偏向群体攻击，单体首领偏向单体攻击。治疗能力必须同时满足“自然法师职业”和“已读取到治疗技能”，其他职业即使旧资料中存在治疗数值也按 0 处理，不会推荐治疗技能。作用于全队的普通增益仍按辅助处理，不会误判为治疗。资料不足时只显示当前职业适用的技能类别，不会虚构具体技能。

### 第七步：发布正式方案

确认方案后点击“发布方案”。

只有已经上传过资料、且能与远程成员编号匹配的会员会收到个人正式分配。发布后通知会员点击“只查看最新分配”。

## 8. 高级维护：成员原始表格

正常使用通过“成员”页的选择框配置，无需编辑原始表格。只有批量修正评分或排查数据时才需要展开本节。

CSV 第一行必须是表头，`name` 必填。常用字段：

| 类别 | 字段 |
| --- | --- |
| 基础 | `name`, `level`, `lifeLevel`, `combatLevel` |
| 当前报名 | `currentLife`, `currentCombat` |
| 生活评分 | `milking`, `foraging`, `woodcutting`, `cheesesmithing`, `crafting`, `tailoring`, `cooking`, `brewing`, `alchemy`, `enhancing` |
| 战斗评分 | `combat`, `aoe`, `single`, `physical`, `magic`, `stab`, `slash`, `blunt`, `ranged`, `tank`, `healer`, `sustain`, `support`, `power` |
| 分类 | `role`, `damageType`, `fixedCombat` |
| 偏好 | `preferLife`, `preferCombat`, `avoid` |

评分数值越高，分配优先级越高。`healer` 只对 `weaponType=nature` 生效；普通成员窗口也只会向自然法师显示“职责：治疗”。

`preferLife` 和 `preferCombat` 可以填写英文 key 或中文试炼名。`avoid` 可使用英文逗号或竖线分隔多个不想参加的试炼。

`role` 支持 `debuff` 或“减益”。`fixedCombat` 是硬约束，可填 `badger`、`chameleon`、`jellyfish`、`hedgehog`、`swarm` 或对应中文名；固定位置优先于 `avoid`。

## 9. 数据与安全

### 9.1 上传内容

会员端上传经过裁剪的角色摘要，不上传：

- 游戏登录凭据。
- 食物和饮料。
- 背包全部物品。
- 成就和无关角色数据。

### 9.2 存储方式

会员资料以明文 JSON 保存到 GitHub 私有仓库：

```text
guilds/<公会ID>/weeks/<周ID>/members/<成员ID>.json
```

拥有该私有仓库读取权限的人可以看到这些资料。

### 9.3 令牌规则

- `GITHUB_TOKEN` 只保存在 Vercel 环境变量中，不进入油猴脚本。
- `LEADER_TOKEN` 只交给会长和授权管理人员。
- 会员端不使用令牌，只按公会编号和当前角色名匹配本周名单。
- PAT 或会长令牌泄露后应立即更换。

姓名匹配不是强身份认证。懂接口的人可以伪造名单中的角色名并覆盖该角色资料，也可以读取该角色的正式分配。需要防止冒用时，应恢复签名会员令牌或接入游戏官方可验证身份。

更换 Vercel 环境变量后必须重新部署，旧部署不会自动获得新值。

## 10. 常见问题

### 10.1 “尚未捕获登录角色数据”

刷新游戏页面，等待角色完全加载后再点击“一键上传并查看分配”。如果仍失败，可在“连接设置”中点击“只读取本角色”，或在高级维护中导入角色原始数据。

### 10.2 读取到的试炼与实际不一致

必须停留在公会“试炼”页面，并确保 4 场生活和 2 场战斗都已加载。刷新页面后重新点击“读取试炼”。

### 10.3 成员等级读取不到

- 先打开公会“成员”页面再点击“读取成员”。
- 公开资料需要实际打开成员资料页，再点击“读取已打开的资料”。
- 隐藏资料或游戏未推送的字段需要会员端主动上传。

### 10.4 会员提示“不在同步的公会名单中”

检查会员端公会编号是否与会长端一致，并确认会长已在本周读取成员后点击“同步本周名单”。姓名匹配忽略大小写和空格，但不会猜测不同拼写。

### 10.5 会长操作返回 401

检查会长端设置中的令牌是否与 Vercel `LEADER_TOKEN` 一致。不要填写 GitHub PAT。

### 10.6 GitHub 写入返回 403

检查 fine-grained PAT：

- Resource owner 正确。
- 只选择仓库 `-mwi-guild-trial-data`。
- `Contents` 权限为 `Read and write`。
- PAT 未过期、未撤销。

### 10.7 健康检查正常，但修改变量后功能仍旧异常

Vercel 环境变量只对新部署生效。修改变量后执行 Redeploy，再测试正式域名。

### 10.8 请求跳转到 Vercel 登录页

项目启用了 Vercel SSO Deployment Protection。油猴脚本无法完成交互式 Vercel 登录，需要关闭该项目的 SSO 部署保护。应用自身的会员和会长 Bearer Token 鉴权仍会保留。

### 10.9 生成方案后大量成员未分配

依次检查：

1. “成员”页是否有成员姓名。
2. 页面是否读取到实际试炼名额。
3. 是否错误勾选“只给未报名成员分配”。
4. 会员上传是否已拉取。
5. `avoid` 是否排除了过多试炼。
6. 高级维护中的首领属性权重是否能正常解析。

## 11. 推荐的公会管理流程

```text
周五重置后读取试炼
  -> 读取成员和当前报名
  -> 同步本周名单
  -> 会员一键上传角色资料
  -> 会长拉取会员资料
  -> 补充缺失资料
  -> 生成并审核分配
  -> 发布方案
  -> 会员查看最新分配
  -> 按推荐方向在游戏中手工报名
```

插件只提供辅助推荐和信息汇总，最终报名与调整仍由公会管理人员和成员在游戏内完成。
